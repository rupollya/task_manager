Task Manager

Task Manager- это инструмент позволяющий эффективно организовывать и отслеживать выполнение задач. Он предоставляет пользователям возможность создавать и редактировать свои задачи, устанавливать сроки выполнения и отслеживать прогресс.

Спецификация проекта

Общее описание проекта.

  • Название проекта: менеджер задач.

  • Цель проекта: разработать веб-сайт для управления задачами в разных категориях.

Требования к проекту.

  1. Проект должен работать на движке Starlette и шаблонизаторе Jinja.

  2. Иметь гибкий интерфейс.

  3. Иметь аутентификацию на основе механизма сессий.

Установка и настройка

На локальном сервере, для личного использования

Для запуска проекта требуется установить databases, asyncpg, sqlalchemy, starlette, jinja2, passlib, pyjwt, uvicorn.

Установка выполняется в терминале.

pip install (название програмного модуля)

После успешной установки создать базу данных, например в pgAdmin 4.

  • Имя базы данных - task_manager

  • Таблицы CREATE TABLE Users (user_id SERIAL PRIMARY KEY,login VARCHAR(20) UNIQUE, password VARCHAR(16)); и CREATE TABLE Task ( task_id SERIAL PRIMARY KEY, user_id INTEGER, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, important BOOLEAN DEFAULT FALSE, completed BOOLEAN DEFAULT FALSE, heading TEXT, task_text TEXT, data_stop TIMESTAMP, prize TEXT, FOREIGN KEY (user_id) REFERENCES Users(user_id) );

Запустить проект .

Python3 -m uvicorn main:app --reload

Здесь main - название основного файла.

На удаленном сервере, как для личного, так и для не личного использования

Требуется загрузить проект на github, выложив на него все файлы, не забыв про файл зависимостей - requirements.txt.

Его содержимое: databases asyncpg sqlalchemy starlette jinja2 passlib pyjwt uvicorn.

После развернуть проект, например на Render.

Там же создать базу данных, к которой можно подключиться через pdAdmin 4 и настроить ее для проекта.

В итоге должно получится так.

Основной функционал проекта

Создание и редактирование задач

Пользователи могут создавать новые задачи, задавая им заголовок, описание и другие параметры(ниже сказано). Редактирование задач позволяет пользователям обновлять информацию о задачах, включая изменение заголовка, описания, тегов и сроков выполнения. Пример работы * Создание задачи: пользователь заполняет форму с полями заголовка и описания,выбирает дату завершения, награду за выполнение и теги, после чего задача сохраняется в базе данных. * Редактирование задачи: пользователь выбирает задачу из списка и изменяет необходимые поля, после чего изменения сохраняются в базе данных.

Удаление задач

В рамках проекта, задачи можно не удалять навсегда, им можно присвоить тег - завершенный. Такой подход позволяет поддерживать актуальность и чистоту списка задач. Либо же пользователь может удалить задачу навсегда, если она не несет никакой ценности. Пример работы

  • Пользователь завершил задачу, зашел в режим редактирования и поставил тег - завершенный. Теперь задача будет хранится в списке завершенных задач.

  • Удаление задачи: пользователь выбирает задачу и удаляет ее. Задача удаляется из базы данных.

Тегирование задач

Пользователь может добавить теги к задачам для их классификации и облегчения поиска. Пример работы

  • Пользователь считает, что задача на данный момент является важной и ставит ей тег - важная задача, либо как описано выше, можно отнести задачу к завершенным, чтобы не удалять ее навсегда.

Датирование задач

Задачи имеют дату начала - дата создания задачи, и дату, когда задача должна быть выполнена.

Поиск задачи по заголовку

Пользователи могут искать задачи по заголовкам, вводя в поиск любую букву содержащуюся в задаче. Пример работы

  • Пользователь хочет найти задачи связанные с програмированием, он вводит в поиске "програ" и ему выдаются все соответствия.

Авторизация и регистрация

Приложение поддерживает регистрацию новых пользователей и авторизацию существующих. Этот функционал обеспечивает безопасность данных(хэширование паролей) и индивидуальный доступ к задачам. Пример работы

  • Регистрация: новый пользователь заполняет регистрационную форму, после чего его данные сохраняются в базе данных, и ему предоставляется доступ к системе.

  • Авторизация: существующий пользователь вводит свои логин и пароль для входа в систему. При успешной авторизации пользователь получает доступ к своим задачам.

Руководство пользователя

Инструкция по использованию проекта

Перед началом работы с проектом, необходимо зарегистрироваться.

Вы должны придумать уникальный логин и пароль, где пароль состоит не менее чем из 8 символов. После успешной регистрации, вы можете авторизоваться и войти в проект.

Без лишних слов перейдем к созданию своей первой задачи, нажав на кнопку добавить задачу.

Заполним поля и выберем дату завершения задачи. Добавим себе какую-нибудь награду за успешное выполнение задачи. Отнесем задачу в важное.

Сохраним задачу, нажав кнопку сохранить задачу и посмотрим, что изменилось.

Наша задача появилась как в 'важное', так и в "все задачи","сегодня". Кликнув по задаче можно просмотреть ее содержимое, оно отобразится в правой части экрана.

Для редактирвоания задачи следует нажать значок редактирвоания - гаечный ключ. Чтобы удалить задачу - мусорку. Имея много задач можно искать нужную задачу по поиску(левый верхний угол).
